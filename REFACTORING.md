# REFACTORING.md — План переработки сайта под требования IT-аккредитации

## 1. Цель

Переработать существующий сайт `untillpro.github.io` (Jekyll + тема Spectral) в
минималистичный русскоязычный Landing Page для **ООО «Программные решения
Антилл»**, отвечающий требованиям Минцифры для аккредитации IT-компаний:

- описание IT-услуг и ПО;
- реквизиты компании;
- контактные данные;
- информация о продуктах;
- описание технологий и направлений деятельности.

Сохраняется текущий визуальный layout (Spectral / one-page), переводится контент
на русский язык, удаляется всё, не относящееся к требованиям.

---

## 2. Итоговая структура страницы (`index.html`)

Одностраничный сайт со следующими секциями (сверху вниз):

1. **Banner** — название компании, краткое позиционирование, CTA «Связаться».
2. **О компании / Направления деятельности** — кто мы, чем занимаемся
   (разработка ПО для HoReCa, входим в международную группу unTill).
3. **Продукты и IT-услуги** — что мы разрабатываем и предоставляем:
   - unTill Prime — POS-система для ресторанов и гостиниц;
   - unTill Air — облачная POS-система для малого бизнеса;
   - unTill Payments — интегрированный платёжный модуль;
   - unTill WebManagement — облачная панель управления;
   - заказная разработка ПО для HoReCa и сопутствующая интеграция.
4. **Технологии** — стек и направления НИОКР (Go, TypeScript/React,
   PostgreSQL, Kubernetes, облачные сервисы, безопасность платежей и т.д.).
5. **Контакты и реквизиты** — юр. наименование, ИНН, КПП, ОГРН, юр. адрес,
   фактический адрес, телефон, e-mail.

Секции _News_, _CTA «This is your chance»_, _иконки с Lorem/Ipsum/Dolor_ —
**удаляются**.

## 3. Изменения по файлам

### 3.1. `_config.yml`

- `title: "ООО «Программные решения Антилл»"`
- `description: "Разработка программного обеспечения для индустрии гостеприимства (HoReCa)"`
- `owner: "ООО «Программные решения Антилл»"`
- `email: info@untill.com` — оставить.
- `lang: ru` — добавить (используется в `<html lang>`).
- Блок `social:` — очистить (оставить пустым или удалить целиком),
  т.к. публичные соцсети у юр. лица для целей аккредитации не требуются.

### 3.2. `_includes/head.html`

- `<html lang="ru">` (через переменную `site.lang`).
- Удалить ссылку на RSS (`<link rel="alternate" type="application/rss+xml">`),
  т.к. блог удаляется.

### 3.3. `_includes/header.html`

- Удалить пункт меню **Log in** (ссылка на `dev.untill.com/sigma/`).
- Удалить пункт **RSS Feed**.
- **Burger-меню:** в текущей теме Spectral навигация открывается через
  выезжающую панель (`#menu` с классом `menuToggle`). Для минималистичного
  лендинга на одну страницу есть два варианта:
  - **(А, рекомендуется)** Заменить burger-меню на горизонтальные якорные
    ссылки в шапке: «О компании», «Продукты», «Технологии», «Контакты».
    Минус — потребуется небольшая правка стилей навигации.
  - **(Б)** Полностью убрать `<nav id="nav">` из `header.html`. Шапка
    остаётся только с названием компании; навигация не нужна, т.к. весь
    контент помещается на одном экране со скроллом.
    Финальный выбор — за заказчиком; по умолчанию реализуем вариант **А**.

### 3.4. `_includes/footer.html`

- Удалить блок `ul.icons` (соцсети) — не используется.
- Оставить копирайт с полным наименованием юр. лица.
- Удалить строки «Design: HTML5 UP» и «Built with: Jekyll» (по желанию;
  для аккредитации не требуется, но юридически допустимо оставить).

### 3.5. `index.html`

Полностью переписать содержимое в соответствии со структурой из п. 2.
Использовать существующие классы Spectral
(`wrapper style1 special`, `wrapper alt style2`, `spotlight`, `features` и т.п.)
без правок CSS.

### 3.6. Удаляемые файлы

- `_posts/2014-09-23-Austria-2.md`
- `_posts/2016-05-20-Finland-2.md`
- `assets/austr/`, `assets/fin/` — фото с конференций.
- `_layouts/post.html`, `_layouts/post-dark.html` — больше не используются.
- `_includes/disqus.html` — комментарии не нужны.
- `feed.xml` — RSS блога не нужен.
- `elements.html`, `generic.html` — демо-страницы темы.
- `images/peter.jpg`, `dock.jpg`, `hurdle.jpg`, `toppic.jpg`, `banner~.jpg`,
  `pic01..pic05.jpg` — оставить только те, что реально используются на
  переработанной странице (например, `fuchik.jpg` для блока контактов).
- `README.txt`, `LICENSE.txt` — дубли (есть `.md`-версии).
- Папка `docs/` — относится к стороннему React-приложению и не имеет
  отношения к лендингу. **Проверить у владельца** перед удалением.

### 3.7. Файлы темы (без изменений)

- `_sass/`, `css/main.scss`, `js/*`, `fonts/*` — не трогаем.
- `CNAME` — оставить (`www.dev.untill.com`).
- `favicon.ico` — оставить (при желании заменить на корпоративный).

## 5. Порядок работ

1. Согласовать с заказчиком тексты и реквизиты (раздел 4).
2. Внести правки в `_config.yml`, `_includes/*`, `index.html`.
3. Удалить файлы из п. 3.6.
4. Локальная сборка: `bundle exec jekyll serve` — визуальная проверка.
5. Проверка адаптивности (xlarge → xsmall, существующие брейкпоинты темы).
6. Деплой через GitHub Pages (push в основную ветку).
7. Передать ссылку юристу для финальной проверки перед подачей заявления
   на аккредитацию.

## 6. Что сознательно НЕ делается

- Не меняется визуальная тема Spectral (цвета, шрифты, анимации).
- Не добавляется CMS, формы обратной связи, аналитика — для аккредитации
  это не требуется и увеличивает поверхность для замечаний (152-ФЗ и т.п.).
- Не трогается папка `docs/` (стороннее приложение) до подтверждения владельца.
